Q01. Javaのコレクションフレームワークに関する説明として、最も適切なものはどれですか？
         次の４つから正しい答えを選びなさい。
選択肢：
a. `Collection`は、キーと値のペアを格納するためのインターフェースである。
b. `List`インターフェースは要素の重複を許さず、順序も保証しない。
c. `Set`インターフェースは順序付けられた要素のコレクションであり、インデックスによるアクセスを許可する。
d. `Map`インターフェースは、オブジェクトをキーと値のペアとして関連付けて格納する。キーの重複は許されない。

答え：
d
解説：
`Map`インターフェースは、一意なキーとそれに対応する値をペアで管理します。`Collection`は単一の要素の集まりを扱うインターフェースです。`List`は順序付けられたコレクションで、要素の重複を許します。`Set`は要素の重複を許さない集合ですが、順序は保証されるとは限りません（`HashSet`は順序を保証せず、`TreeSet`や`LinkedHashSet`は順序を保証します）。

Q02. `ArrayList`と`LinkedList`の主な違いに関する説明として、正しいものはどれですか？
         次の４つから正しい答えを選びなさい。
選択肢：
a. `ArrayList`は要素の追加・削除が高速で、`LinkedList`は要素のランダムアクセス（インデックス指定での取得）が高速である。
b. `ArrayList`は内部的に配列を使用しており、要素のランダムアクセスが高速である。`LinkedList`は内部的に要素を数珠つなぎ（双方向リスト）で保持しており、要素の追加・削除が高速である。
c. `ArrayList`は`null`値を格納できないが、`LinkedList`は格納できる。
d. `LinkedList`は`List`インターフェースを実装しているが、`ArrayList`は実装していない。

答え：
b
解説：
`ArrayList`は配列ベースのため、インデックスを使ったアクセス（`get(index)`）はO(1)で非常に高速ですが、要素の追加・削除は後続要素のシフトが必要なため遅くなることがあります。一方、`LinkedList`はノード間のリンクを変更するだけで済むため、要素の追加・削除は高速ですが、特定のインデックスにアクセスするには先頭または末尾から順にたどる必要があり低速です。両者とも`List`インターフェースを実装しており、`null`も格納できます。

Q03. ジェネリクス（Generics）を使用する主な利点は何ですか？
         次の４つから正しい答えを選びなさい。
選択肢：
a. プログラムの実行速度を大幅に向上させる。
b. コンパイル時に型チェックを行い、意図しない型のオブジェクトがコレクションに混入するのを防ぎ、実行時エラー（`ClassCastException`）のリスクを減らす。
c. コレクションに格納できる要素の数を増やす。
d. すべてのオブジェクトを`Object`型として扱うことで、コードの記述を簡略化する。

答え：
b
解説：
ジェネリクスの最大の利点は、コンパイル時の型安全性（type safety）の向上です。`ArrayList<String>`のように型を指定することで、そのリストには`String`型のオブジェクトしか追加できなくなり、コンパイラがそれを保証します。また、要素を取り出す際にキャストが不要になるというメリットもあります。

Q04. `HashSet`と`TreeSet`に関する次の説明のうち、正しいものはどれですか？
         次の４つから正しい答えを選びなさい。
選択肢：
a. `HashSet`は要素の順序を保証しないが、`TreeSet`は要素が追加された順序を保証する。
b. `HashSet`は要素の重複を許すが、`TreeSet`は許さない。
c. `HashSet`は`hashCode()`と`equals()`メソッドを利用して要素を管理する。`TreeSet`は要素が自然順序付けされているか、`Comparator`が提供される必要があり、要素をソートされた順序で保持する。
d. `TreeSet`は`null`要素を許すが、`HashSet`は許さない。

答え：
c
解説：
`HashSet`はハッシュテーブルを使っており、高速な検索・追加・削除を提供しますが、順序は保証されません。要素の同一性は`hashCode()`と`equals()`で判断します。`TreeSet`は赤黒木（Red-Black Tree）で実装されており、要素を常にソートされた状態で保持します。そのため、要素は`Comparable`インターフェースを実装しているか、`TreeSet`のコンストラクタに`Comparator`を渡す必要があります。`HashSet`は`null`を一つだけ許容しますが、`TreeSet`は（通常）`null`を許容しません。

Q05. `ArrayList`を使って、文字列のリストを作成し、以下の操作を行うコードを記述してください。

操作：
1. "Apple", "Banana", "Cherry" の3つの文字列をリストに追加する。
2. リストの2番目の要素（インデックス1）を "Blueberry" に置き換える。
3. リストの最初の要素を削除する。
4. 拡張for文を使って、最終的なリストのすべての要素をコンソールに出力する。

答え：
```java
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> fruits = new ArrayList<>();

        // 1. 要素の追加
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");

        // 2. 要素の置き換え
        fruits.set(1, "Blueberry");

        // 3. 要素の削除
        fruits.remove(0);

        // 4. すべての要素を出力
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```
解説：
`ArrayList`のインスタンスをジェネリクス`List<String>`で作成します。要素の追加には`add()`、置き換えには`set(index, element)`、削除には`remove(index)`メソッドを使用します。最後に拡張forループでリスト内の各要素を順に表示します。最終的なリストは ["Blueberry", "Cherry"] となり、これが順に出力されます。
